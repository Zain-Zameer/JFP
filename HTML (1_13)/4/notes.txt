Because of core web vitals.

CLS: (Cummulative Layout Shift)
CLS should be less and less, why? here is the example you can watch: https://web.dev/articles/cls
CLS means ke page load hote hote kitna shift horaha hai.
lets say koi content likha tha, achanak se image load huwi or woh content neeche hogya, a bad user experience right? so agr ap images mein width or height lagate hein toh layout shift nh hoga. makesure to add it inside html

LCP: (Largest Contentful Paint) 
joh sbse bara dikhne wala element hai woh kitni der mein load horaha hai inside a web page. a good LCP score is 2.5 seconds or less.

FID: (First Input Delay)
Agr user ne web page pr koi input dia (koi data enter kia ya koi subimit button pr click kia kisi bhi trha ka input) toh woh process hone mein kitna time lagra hai. A good FID is 100 milisconds or less.
